List of Metadata Entries

 

The list of metadata entries can be both edited as well as expanded. This means you can modify the name and content of a metadata entry, as well as append a new entry to the list. Each entry in the metadata consists of three parts, namely:

  1. A checkbox: If checked, the entry is included in the exported DTB.
  2. Name of the entry
  3. Content of the entry

 

The list is sorted in the above order; all the checked entries will appear in the beginning. Then, the checked entries will be sorted by name, and then by content. Similar action will be taken for the unchecked entries.

 

You can select an entry by clicking it using the mouse or by up and down arrow using the keyboard.

 

Please note that all the unchecked entries of the list belong to the Dublin Core metadata or X-metadata.

 

The following table provides information about the metadata entries.

Name of Entry

Occurrence

Description

dc:Identifier

Required

A string or number identifying the DTB. This identifier should be compliant with the scheme of allotting identifiers to a book in your organization.

dc:Language

Required

Language of the content of the publication. For example: Type EN-US for English of United States.

dc:Publisher

Required

The agency responsible for making the DTB available.

dc:Title

Required

The title of DTB including any subtitles.

dtb:narrator

Required

Name of the person who narrated the book.

Generator

Required

The software that generated the book, Obi, in this case.

Obi:xuk version

Required

Version of the xuk file (project file with an .obi extension).

dc:Contributor

Optional

A party whose contribution to the publication is secondary to those named in dc:Creator.

dc:Coverage

Optional

The extent or scope of the content of the resource. Not expected to be used for DTBs.

dc:Creator

Recommended

Names of primary author or creator of the intellectual content of the publication.

dc:Date

Required

Date of publication of the DTB. Syntax is YYYY[-MM[-DD]] with a mandatory 4-digit year, an optional 2-digit month, and, if the month is present, an optional 2-digit day of month.

dc:Description

Optional

Plain text describing the publication's content.

dc:Relation

Optional

A reference to a related resource.

dc:Rights

Optional

Information about rights held in and over the DTB.

dc:Source

Recommended

A reference to a resource from which the DTB is derived. For example: A print original, ebook, etc. Best practice is to use the ISBN when available.

dc:Subject

Recommended

The topic of the content of the publication.

dc:Type

Optional

The nature of the content of the DTB (i.e., sound, text, image). Best practice is to draw from the Dublin Core's enumerated list [DC-Type].

Dtb:Produce Date

Optional

Date at which the book is produced. Format will be similar to dc:Date.

Dtb:Producer

Optional

The name of organization or library that published the DTB.

Dtb:Revision

Optional

The number indicating the revision of the book, if it has been revised.

Dtb:Revision Date

Optional

Date at which book is revised.

Dtb:Revision Description

Optional

A brief description about the modifications.

Dtb:Source Date

Optional

The date of publishing mentioned on the physical book.

Dtb:Source Edition

Optional

Edition of the physical book.

Dtb:Source Publisher

Optional

Publisher of the physical book.

Dtb:Source Rights

Optional

The name of the organization that holds the rights of the physical book.

Dtb:Source Title

Optional

Title of the physical book.

Custom

 

Provided to create a new metadata entry

 

Please note that the metadata entries can be repeated.

 

It is recommended to fill in as much metadata as you can before publishing the book. The required metadata entries are checked for null references before exporting.

 

For a detailed information on metadata entries, please visit http://www.daisy.org/z3986/2005/Z3986-2005.html#PubMed.
