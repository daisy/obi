Importing and Replacing TOC

Obi enables you to import the structure from an XHTML, CSV or dlimited text file and replace the existing TOC of your project. This allows narrator to record the contents without bothering about the structure of the book. The contents can be recorded linearly without giving the proper name or level to the corresponding sections. Everything can be brought into order by importing TOC and replacing the existing one by a single command.

The process is exemplified in the following steps.

  1. Record the contents of the book in your Obi project by creating a new section for each chapter, topic and sub topic.
  2. Create the structure of the book in an XHTML, CSV or delimited text file.
  3. Open your project in Obi and click Sections->Import and replace TOC on the menu. Obi will open a file browser asking you to select the file.
  4. Select the file containing the structure of the book and click on the Open button.

Obi will not only replace the section names but also adjust the levels of the sections according to the corresponding tags. But for this functionality to work properly, it is very important to record the sections in the correct order.

Let's look at the following example from an XHTML file.
<h1>Micro Economics</h1>
<h2>Inflation</h2>
<h2>Demand and Supply</h2>

While importing the TOC, Obi will do the following:
“Micro Economics” will become the first section in the project at level 1.
“Inflation” will become the second section of the project and will be at level 2.
“Demand and Supply” will be the third section of the project and will be at level 2.
