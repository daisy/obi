Creating a Project by Importing DAISY Book

 

Obi enables you to create a project by importing the structure and audio from an existing DAISY book. The formats expected by DAISY book import are:

  • DAISY 3 audio only
  • DAISY 3 full-text full-audio
  • DAISY 2.02 audio only
  • DAISY 2.02 full-text full-audio
  • DTBook.xml

 

Follow the steps listed below to create a project by importing a DAISY book:

1.      Click File->New Project from Import on the menu or press Ctrl+Shift+N. an Open dialog pops up that lets you browse for your DAISY book.

2.      Select the manifest file (with .opf extension) for DAISY 3 book, NCC document (with .ncc extension) for DAISY 2.02 book, or XML file for DTBook.

3.      Click on the Open button or press Enter. A dialog box opens as shown in the figure below:

 

Figure: Create a new project from Import

 

 

4.      Enter the following details in the dialog: 

Field

Field Type

Description

Title

Edit Box

Obi extracts the title from the DAISY book automatically. However, you can change it.

ID

Edit Box

Enter a unique identifier for the book. This identifier should be compliant with the scheme of allotting identifiers to a book in your organization. You can leave this field with the default value provided, if you are not sure at the moment. ID is stored in the metadata and can be changed any time.

Generate

Button

Generates an identifier for your project randomly and fills in the ID field.

Location

Edit Box

Enter the location where you want to store your project. You can also browse for a location by Clicking the Select button.

Select

Button

Opens a dialog box to select a directory for your project.

If the selected location is not empty, Obi will suggest one. Click Yes to accept or click No to ignore.

 

1.      Click on the OK button. Obi will display a dialog box to supply the audio settings for the project.

2.      Enter appropriate values and click on the OK button.

 

Obi will obtain the structure and audio from the selected DAISY book and create a new project in congruence with acquired data. After completion, a dialog box will be displayed reporting the result of the operation along with the list of errors, if there were any. If the errors were due to mismatch of the duration of audio content of a phrase and its entry in the SMIL file, you can adjust the import tolerance using the Project preferences tab of the Preferences dialog and try to import the DAISY book again.

 

The application window of Obi will then display the sections and their contents, the title bar indicating the title extracted from the DAISY book.

The TOC view will exhibit the imported sections with the same indentation as they had in the DAISY book and the Content view will expose the contents of the first section of the project.

 

Please note that Obi may not import all the elements of the DAISY book. What will be retained or discarded depends on the format of the selected DAISY book. The table below will suggest you what to expect in your project after the import operation.

 

DAISY book format

What will be imported?

What will be discarded?

DAISY audio only

Entire structure and audio

Nothing

DAISY 3 full-text full-audio

The structure and audio

The text not representing the structure elements

DTBook.xml

The structure of the book

The text not representing the structure elements

Remarks:

The structure elements imported in this operation are:

  • Section names and their depth
  • Heading phrases
  • Page phrases
  • Skippable notes
