Creating a Project by Importing an xhtml file

If you have an xhtml file corresponding to the book, you can import the structure of the project from it. Follow the steps given below to create a new project by importing an xhtml file:

1.      Click File->New Project from Import on the menu or press Ctrl+Shift+N. an Open dialog pops up as shown in the figure below:

Figure: Choose Import File

 

2.      Select an xhtml file and click Open. A dialog box opens as shown in the figure below:

Figure: Create a new project from XHTML Import

 

 

3.      Enter the following details in the dialog: 

Field

Field Type

Description

Title

Edit Box

Obi extracts the title from the xhtml file automatically. However, you can change it.

ID

Edit Box

Enter a unique identifier for the book. This identifier should be compliant with the scheme of allotting identifiers to a book in your organization. You can leave this field with the default value provided, if you are not sure at the moment. ID is stored in the metadata and can be changed any time.

Generate

Button

Generates an identifier for your project randomly and fills in the ID field.

Location

Edit Box

Enter the location where you want to store your project. You can also browse for a location by Clicking the Select button or pressing Alt+S.

Select

Button

Opens a Windows file browser to select a directory for your project.

If the selected location is not empty, Obi will suggest one. Click Yes or press Alt+Y to accept or click No or press Alt+N to ignore.

 

1.      Click OK or press Alt+O to create the project. Obi will attempt to create the specified directory, if it does not exist.

 

A new project opens in the work area of Obi and the TOC view shows the sections and subsections Imported from the xhtml file.

 

The level of the section will depend on the heading number in the xhtml file. For example, the text bounded by the tags <h3> and <\h3> in the xhtml file will become the section name of the section at level 3.

 

The Content View shows an empty strip corresponding to each section or subsection in the TOC View. Obi also creates a phrase, without an audio, corresponding to each page in the xhtml file and assigns a page role to it.

 

The page kind and the page number will be determined by the tags corresponding to the pages in the xhtml file. For example, <span class=”page-front” 52 ></span> will become a page-front with page number 52.

 

The next important thing is the position of the page. Obi attaches the page to the section just created.

 

For example:

Consider the following tags

<h3> Inflation </h3>

<span class=”page-normal”> 22 </span>

Obi will create a section named Inflation at level 3. Then, it will create a page phrase under this section with page number 22 of page-normal kind.

 

 

Format expected by Obi

The tags expected by Obi to identify headings and pages are listed as follows:

  • Headings

<h1>…<\h1>: Becomes a section at first level.

<h2>…<\h2>: Becomes a section at second level.

<h3>…<\h3>: Becomes a section at third level.

<h4>…<\h4>: Becomes a section at fourth level.

<h5>…<\h5>: Becomes a section at fifth level.

<h6>…<\h6>: Becomes a section at sixth level.

 

  • Pages

<span class=”page-front”>#<\span>: Becomes a phrase tagged as page-front. The # becomes the page number of the page phrase.

<span class=”page-normal”>#<\span>: Becomes a phrase tagged as page-normal. The # becomes the page number of the page phrase.

<span class=”page-special”>#<\span>: Becomes a phrase tagged as page-special. The # becomes the page number of the page phrase.

 

For more information on sections, see Working with Sections.

For more information on pages, see Pages and Special Roles.
