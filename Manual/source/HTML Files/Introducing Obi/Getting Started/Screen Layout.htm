Screen Layout

Obi has a meticulously designed screen that helps you manage the contents of the book in a structured and efficient way.

The Obi screen is divided into various segments, which are separated by splitters. They are:

  • TOC View
  • Content View
  • Metadata View
  • Transport Bar
  • Status Bar

When you open a project in Obi, all the views, except the Metadata view, are visible. These views can be shown or hidden by using the menu bar or keyboard shortcuts, except for the Content View. The Content View is always visible.

The figure below illustrates how Obi screen looks:

TOC View

TOC View or Table of Contents View, as the name suggests, shows the table of contents of the book. It provides a list of all the sections (book-parts, chapters, topics and sub-topics etc) of the BOOK IN a HIERARCHICAL WAY (similar to a tree structure). A section or a subsection is represented by a single node of the tree, and its indentation indicates the level of the section.

Position

TOC View is positioned on the top left side of the Obi screen.

Content View

Content view exposes the phrases of a section in horizontal lists. The visual representation of a phrase comprises of a phrase label and a waveform representing the audio contained in it. This view is designed to perform all the audio insertion and editing.

Position

Content View is positioned on the top right side of the Obi screen.

Metadata View

Metadata View shows the metadata of the book in the form of a list. It contains the information such as title, narrator, date of publication, etc.

It also has an edit area that allows you to modify a metadata entry or create a new one.

Position

Metadata View is positioned on the left side of the Obi screen under the TOC View.

Transport Bar

Transport Bar shows a list of buttons lined horizontally. These buttons are used for recording, playback and navigation. There is a time display showing the elapsed or remaining time during the playback or recording. It also shows the textual peak meter for audio monitoring.

Position

Transport Bar is positioned on the bottom of the Obi screen above the Status Bar.

Status Bar

Status Bar is a read only section of the Obi screen showing the current state of Obi.

Position

Status Bar is the bottom most area of the Obi screen.

External Views

Obi also provides three external views:
  • Recording toolbar
  • Graphical Peak Meter
  • Source View

Recording toolbar

Recording toolbar is a comprehensive pop-up control bar for live recording. It offers all the controls mandatory for recording and structure mark-up with only a small toolbar visible on the screen.

Graphical Peak Meter

Graphical Peak Meter shows the amplitude of the audio currently being played or recorded with respect to a single unit of time. It also shows the overloads during playback or recording, if any.

Source View

Allows you to view or modify the project file. This feature should be used only if you are thoroughly familiar with the structure of xuk format as mishandling will render the project invalid.

You can navigate among various views of Obi by pressing Ctrl+Tab or Ctrl+Shift+Tab, except for the external views, for which the shortcut keys are Alt+Tab.

While pressing Ctrl+Tab, the focus will reach panel splitter between TOC View and Content View. This splitter is helpful in adjusting the sizes of these views by moving it to left or right. Press Left Arrow for moving to left and Right Arrow for moving to right.
