What is Obi?

Obi is a production tool for DAISY DTBs. It can produce DAISY/NISO 2005 "NCX-Only" DTBs natively.

Obi is free to use, internationalized and accessible. Obi has been designed to be easy to learn and use.

Obi - Installation and system requirements

Obi runs on Windows XP (SP2) and Vista. The software requires the use of Microsoft's .Net and DirectX for audio. This section remains to be written

Obi - Getting Started

This section of the Obi manual contains a "quick start" guide briefly describing how to use Obi. A more detailed description of Obi features and functions will be available in the Obi - Pro Reference section.

Screen Layout

The Obi screen is divided into the following views:

  • Table of Contents (TOC) view (F6);
  • Content View (F6);
  • Transport Bar (Ctrl+F6);
  • Metadata view;

We can navigate among these views by pressing Ctrl+Tab or Shift+Ctrl+Tab. There are splitters between the content view, the TOC view, and the metadata views to change the size of these views.

Step-by-step Guide to DTB Creation with Obi

Creating a New Project

Open Obi, and create a new project (File > New project, or Ctrl+N.) Choose a title for your project, and a location on your system where it will be saved. You can also tell Obi to automatically create a title section using the title of your project.

Every DAISY book must have a unique identifier. Your organization may already have a scheme to allocate unique identifiers to new projects; otherwise, Obi can generate one for you. If you are not sure, you can use the default value provided by Obi. The unique identifier can be set in the textbox labelled ID; you can edit it later by setting the dc:Identifier value in the Metadata view.

Alternatively, you can create a new project by importing an existing structure from an XHTML source document containing headings (h1 through h6) and pages (span); choose File > New project from import or press Ctrl+Shift+N. You can choose a title for your project (which defaults to the title value in the XHTML file) and a location for saving it.

You can save your project at any time (File > Save project or Ctrl+S.) There is currently no auto-save in Obi so you should save your changes often to minimize the risk of losing important data. You can then reopen your project at a later time by choosing File > Open project or pressing Ctrl+O. It is also possible to configure Obi to automatically reopen the last open project when starting, or to quickly find recently open projects through the File > Recent projects menu item (Alt+F, R.)

Editing the Table of Contents

You can then create or modify the main structure elements in the TOC view by the following commands:

  • Adding a new section (Sections > Add section, or Ctrl+H) after the selected section, or at the end of the project if no section is selected.
  • Inserting a new section (Sections > Insert section, or Ctrl+Alt+H) before the selected section.
  • Adding a new subsection (Sections > Add subsection, or Ctrl+Shift+H) under the selected section.

When a new section or subsection is created, it appears in both the TOC view (where the TOC is viewed as a tree) and in the content view (as a strip where audio content can be added.) The newly created section is also selected and can be named immediately.

During structure editing, you may also find the following commands useful:

  • Renaming a section (Sections > Rename section, or F2) to edit the heading of a section.
  • Changing the level of a section (Sections > Decrease section level, or Ctrl+Shift+Left; and Sections > Increase section level, or Ctrl+Shift+Right).
  • Basic edit commands such as copy (Edit > Copy, or Ctrl+C), paste (Edit > Paste, or Ctrl+V;Edit > Paste before, or Ctrl+Alt+V; and Edit > Paste inside, or Ctrl+Shift+V), delete (Edit > Delete, or Del) and cut (Edit > Cut, or Ctrl+X.) Copying and deleting affect the whole selected section, including its subsections.

Adding and Editing Audio Content

After creating sections, audio can be added to these sections either by live recording, or by importing existing audio files.

Live Audio Recording

Select the section that you want to record in in the contents view and click on the record button in the transport bar, or press Ctrl+R for starting pre-recording monitoring.

The amplitude of the microphone input will be displayed in the graphical peak meter (which can be shown or hidden by the command View > Show peak meter, or Ctrl+Alt+P) as well as a text peak meter located in the transport bar. You can then change the settings for your microphone through system settings. You should set your recording level so that the audio level is loud enough to be heard comfortably, but not so loud as to create clipping and distortion. You should also try to maintain the volume consistently throughout the book.

When the audio levels are to your liking, click the record button one more time in the transport bar, or press Ctrl+R again to start recording. If you want to skip pre-recording monitoring and start recording directly, you can also press Ctrl+Shift+R. Everytime you start recording, a new phrase is created in the selected section. If nothing is selected, a new section is added to record in, and if a phrase is selected, recording starts after this phrase. You can record before a given phrase by selecting the position before this phrase.

During recording, you can create new phrases, pages and sections on the fly using the transport bar:

  • Press the next phrase button, or K, to create a new phrase and continue recording in this phrase.
  • Press the next page button, or P, to create a new page.
  • Press the next section button, or H, to create a new section.
Importing Audio Files

You can import external audio files into new phrases with the command Phrases > Import audio files (Ctrl+I). You are then presented with a file dialog to choose one or several WAV files (usually having a .wav extension) to import. Note that the files must have the same audio settings as the book (i.e., same number of channels, sample rate and bit depth.) For every selected file, a new phrase is created with the corresponding audio in the selected section or after the selected phrase or position in a strip.

Editing Contents

After recording, contents can be edited more finely. You can navigate through the content view by pressing arrow keys (up and down to move from section to section, while left and right move from phrase to phrase); you can also click on a phrase or a section to select it if you are using a mouse.

Phrases can be split and merged. To split a phrase in two, select a phrase and play it using the play button in the transport bar or by pressing the space bar. Pause at the time where the phrase should be split (by pressing the pause button in the transport bar or the space bar again), then choose Phrases > Split phrase (Ctrl+Q).

A new phrase is created following the current phrase, and the audio following the split point is moved to the new phrase. The converse operation is merging two phrases: select a phrase, then choose Phrase > Merge phrase with next (Ctrl+M) to merge the audio of the selected phrase with the next one into a single phrase.

A related operation is phrase detection, to split a phrase automatically according to silent chunks within the phrase (Phrases > Apply phrase detection, or Ctrl+Shift+P.)

Sections can also be split and merged (Sections > Split section, or Ctrl+Shift+Q; and Sections > Merge section with next, or Ctrl+Shift+M).

Basic edit commands such as copy (Edit > Copy, or Ctrl+C), paste (Edit > Paste, or Ctrl+V), delete (Edit > Delete, or Del) and cut (Edit > Cut, or Ctrl+X.) apply to strips, phrases as well as the waveform of the audio inside the phrases.

Note that in the Content View, the area before and after a phrase can be selected. This is useful when you want to paste some content between two already existing phrases, or when you want to define a section split point.

Pages and Special Roles

Pages can be marked by selecting a phrase and choosing Phrases > Assign role > Page, or Alt+P, A, P. A dialog is then presented to input a page number, with the option to automatically renumber the following pages.

Other roles can also be marked in the same way; for instance, you can set a phrase to be the audio heading for a section by choosing Phrases > Assign role > Heading, or Alt+P, A, H (by default, the first phrase of a section is the audio heading.)

Editing Metadata

Before publishing your project, you should fill out as much of the metadata as possible. First, make sure that the metadata view is visible (View > Show metadata view, or Ctrl+Alt+M.) You can reach it by using Ctrl+Tab.

Exporting to DAISY

Finally, you can create an actual DAISY book by exporting your project to DAISY, using the command File > Export as DAISY (Ctrl+E). Note that Obi uses its own file format (files named with the file extension .obi) which is different from the DAISY format. Obi can only read .obi files, so keep your project file if you want to make further modifications to your project, and don't forget to export as DAISY again to make your book playable by DAISY reading systems.

Useful Shortcut Keys

These will save you a lot of time!

  • Save: Ctrl+S.
  • Save As: Ctrl+Shift+S.
  • Play all: Ctrl+Shift+Space.
  • Play selection, pause, resume: Space.
  • Play faster: F.
  • Play slower: S.
  • Play at normal speed: D. (?Note: fast, slow & normal command working only in content view.)

Obi - Pro Reference

This section of the Obi User Manual is under development. In the meantime, please refer to the Obi Reference Manual for details regarding Obi's features.

Contents View

The Contents View exposes sections contained in the book as well as their contents i.e. phrases.

sections are displayed in the form of strips arranged one under another so that they make a vertical list. You can navigate to the next or previous section using the up and down arrow keys. Use the Up arrow key to go to the previous section, and use the Down arrow key to go to the next section.

Each section exposes its phrases in the form of a horizontal list i.e. phrases are arranged one after another, side by side. You can navigate to the next and previous phrase using the left and right arrow keys. The Left arrow key will take you to the previous phrase, and the Right arrow key will move to the next phrase.

Let's take an example. Assuming that you have the focus on the first section, and you need to navigate to the fourth phrase of the third section, the process followed whill be:

  • Press Down arrow twice so as to move to the third section
  • To move inside this section, to its first phrase, use Right arrow.
  • Use right arrow 3 more times in order to reach the fourth phrase i.e. our destination phrase.

NOTE: You cannot move to the phrases of the next or previous section with Right and Left arrow as these do not cross phrase boundaries.

Moving from Phrases to Sections

While keyboard focus is on a phrase, you can use the Up arrow or Down arrow to move out of a phrase to sections. i.e.:

  • Down arrow: bring keyboard focus out from phrase and lands on next section.
  • Up arrow: bring keyboard focus out from phrase and land on the same section which contained the focussed phrase i.e. selects the whole section.

and on pressing up arrow once again move to previous section.

Other keyboard shortcuts for Navigating in the Contents View include:

  • Home key: move to first phrase in a section
  • End key: move to last phrase in section.
  • Ctrl + home: move to first section of book.
  • Ctrl + End: move to last section of book.

Playback in Contents View

There are two modes of playback in Obi: 1. Play all. 2. Play selection.

Play all can be started using ctrl+shift+space. This mode plays the whole book until the end with the following rules:

  • If nothing is selected, playback starts playing the book from the beginning and plays to the end.
  • If keyboard focus is on any phrase or section, playback starts from this phrase / section and plays to the end of the book.

Play selection can be started using the space bar. The same space bar can be used to resume playback after pause. This mode plays the selected part of the book with the following rules:

  • If a phrase is selected, playback plays only this phrase.
  • If a section is selected, playback starts from the first phrase of the section and plays all phrases contained in this section, but excludes phrases contained in subsections of this section.

When playback is started, an audio cursor can be seen moving through the waveform indicating your play position. This cursor continues to move while Obi is playing the recorded content. It stops when playback is paused and remains at this position until playback is resumed. It disappears when playback is stopped.

Both of the playback modes can be paused and stopped using the same set of keyboard shortcuts, i.e.:

  • Space for pause.
  • Control+space for stop.

Playback with Respect to Navigation

While playback is stopped, you can navigate to any section or phrase in the Contents View using the arrow keys or other Contents View keyboard shortcuts. You can start playback in any of two playback modes. Playback will start according to the above listed rules.

While you are playing back content, you can move to any phrase or section in the Contents View using the Contents View shortcuts. If the phrase / section which recieves focus lies within list of phrases selected for playback, playback will also move to focused phrase/section. i.e. player starts playing the phrase/section which recieves focus. Else if, the phrase/section which recieves focus do not lie in list of phrases selected for playback then playback just stops.

While playback is paused, Obi stops the movement of the audio cursor, and this cursor starts moving again when playback is resumed. If keyboard focus is moved away from paused phrase, then playback moves to stopped state and on giving play command again, playback is started from the phrase/section which recieves focus. i.e. playback resumes from paused position of a phrase if and only if the paused phrase remains selected.

The Preferences Dialog

The preferences dialog contains various settings relating to the project and the system. Obi will remember these settings between sessions. In other words, when you restart Obi or your computer, these settings will remain as you left them.

Project Preferences

Default projects directory
Select a directory on your computers harddrive that Obi will suggest as a project directory when you start a new project.
Default export directory
Select a on your computers harddrive that Obi will suggest as a destination directory when you export your project as a DAISY DTB.
Open last project when starting Obi
If this checkbox is checked, Obi will automatically open the last used project on startup.

Audio Preferences

The audio preferences tab allows to view and modify settings relating to audio recording and audio playback.

Input device name
Set the audio device to use for recording. This is the same as selecting a soundcard input port.
Output device name
Set the audio device to use for playback. This is the same as selecting a soundcard output port.
Sample rate
Select the sample rate to use for recording. This field defaults to 44100, and normally, you can leave it at that. In some circumstances (such as a very large book being recorded) you may want to change this setting to 22050.

Audio channels::

Select whether you want to record in mono (one channel of audio) or stereo (two channels of audio). Normally, DAISY DTBs are recorded in mono.

Noise level:: It allows user to adjust recording settings according to level of background noise.

Audio clues:: Enables/disables audio clues. Audio clues are some audio signals which provide audio feedback for non visual users.

User Profile Preferences

Full name
Name of the user.
Organisation
Organisation of the user.
Language
The language of the user (which is not always the same as the language of the DTB being recorded). Remember that you should use the Metadata view to enter information about the language of the DTBs being produced.
