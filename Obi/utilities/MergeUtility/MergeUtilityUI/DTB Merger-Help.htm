This document is a user’s guide for working with DTB Merger – a utility that merges Digital Talking Books.

 

 

Overview

 

DTB Merger is a new utility shipped with Obi 1.1. It enables you to merge two or more Digital Talking Books (DTBs). It has a very simple and user friendly screen that automatically guides the user through the process of merging the DTBs.

 

The publisher may feel the need of DTB Merger due to many reasons, like if the book comes in three volumes as separate DTBs, but due to demand of users he/she wants to distribute them as a single DTB. This can be done using DTB Merger in few simple steps.

 

Also, if the book is very big, the publisher may want to segregate the work into smaller projects, and then merge the exported DTBs before distribution. This allows the publisher to introduce parallel processing by allocating the work to more than one individual, and thereby accelerating the process of book production to a great extent.

 

DTB Merger Screens

 

DTB Merger has meticulously designed screens that take you through the process of merging in a very convenient way. These screens are described as follows:

  • DAISY Standard Selection Screen
  • The Main DTB Merger Screen

 

DAISY Standard Selection Screen

 

This is the first screen that lets you select the DAISY standard to which the merged DTB should conform. Please note that this utility merges the DTBs following the same DAISY specifications only. This means, if you select DAISY 3 in the dialog box, all the DTBs that you select for merging should be compliant with DAISY 3 standard.

Step by step procedure for merging DTBs

 

  1. Launch DTB Merger by selecting Programs->Obi->DTB Merger from the Start menu.
  2. The DAISY Standard Selection screen will be displayed, where you can select the DAISY standard to which the merged DTB should conform.
  3. Mark your selection by selecting the appropriate radio button and click on the OK button.
  4. The main DTB Merger screen will be shown.
  5. Place the first DTB in the list of DTBs to be merged by clicking on the Add button or by pressing Alt+A.
  6. Use the Add button again to append more DTBs to the list.
  7. As DTB Merger merges the DTBs according to their order in the list of input files list box, you must check the order before clicking on the Merge button. You can reorder the DTBs by using the Move Up button to move up one level, or Move down button to move down one level.
  8. You may also delete a DTB from the list by using the Delete button or by pressing Alt+D.
  9. It is advisable to validate the DTBs before merging. To validate a DTB, select it in the list and click on the Validate button or press Alt+I.
  10. Now choose the Page numbering options. If you have ensured that the existing page numbers will appear correctly in the merged DTB, choose the radio button - Keep existing page numbers. But, if you want DTB Merger to renumber the pages, select the radio button - Auto renumber pages.
  11. Provide the full path of the directory where you want to store the merged DTB by clicking on the Browse button or by pressing Alt+B.
  12. Click on the Merge button or press Alt+M to start merging.

 

The Fruition

 

Obi’s DTB Merger will merge the DTBs provided in the main DTB Merger screen and store it in the output directory specified therein. The merged DTB will be compliant to the DAISY standard selected in the DTB Merger Standard Selection screen.

 

The merger will take place sequentially. This means that the second DTB will be merged to the first one, and then the third one will be appended, and so on.

 

The metadata entries of the first DTB will be copied to the merged DTB. This means that the merged DTB will carry the same unique identifier, title etc as the first DTB in the list.

 

The format of the audio files will not be changed during the merger. For example, if the first DTB had audio files in the wav format and the second one had them in MP3 format, they will still appear in the same way in the merged DTB.

 

Please note that the merged DTB will be compliant fully to DAISY standards only if the original DTBs were validated before the merge took place. However, you can validate the merged DTB using the DTB Validator later.
