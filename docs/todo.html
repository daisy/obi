NO Project cleanup November Marisa
Unreferenced audio data is deleted from the asset manager (this part is handled by the toolkit), then unused nodes and their audio data are deleted from the tree. This clears the undo stack as well and modifies the project.
NO Publish November Marisa
A complete copy of the tree along with all of its audio data is made. Cleanup (toolkit) and pruning (Obi) of the copy is performed. The audio assets are consolidated (one audio file per section; file names in the same order as the sections for sequential playback; toolkit). The copy of the tree is serialized to XUK in memory and the XUK-2-Z transformation is applied to the copy (Obi). As a result, the publish directory contains all the DAISY 3 files and the one audio file per section audio data; the original project and its data are left untouched. Questions: (1) does copying the entire tree work? (2) will cleanup really only affect our copy? (3) when will asset consolidation (one asset per section) work?
NO XUK to Z update November Julien, Marisa
Update the XSLT and transformation code for the XUK to Z export so that all new blocks are correctly handled (pages, skippable/escapable items, headings, NCX images, etc.)
