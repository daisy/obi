Selection and command management

Undo and redo should restore the state of the project to what what it was just before the command was issued (undo) or just after it is issued (redo), so there is a tight relationship between commands and selection. Namely, the commands should store the selection for restoring it when undoing.

Selecting something can itself be a command, and thus can be undone and redone. We would need to test this feature as it may lead to overcrowding the command stack, but it is useful, especially when we deal with editing of wave form.
