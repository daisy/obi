Obi is a DAISY/NISO Digital Talking Book production tool. Obi was created to demonstrate the capabilities of the Urakawa toolkit as well as to provide an easy to use, accessible and internationalized DAISY production tool for everyone. Obi is free software; it is available at no cost and its source code is available for anyone to run, modify and redistribute. Obi is released under the [insert actual licence here.]

Features

Obi is by design a simple tool with a small feature set:

  • Support for audio-only book editing with a table of contents (in DAISY terms, this means NCX-only DTBs.)
  • Support for uncompressed RIFF/WAVE audio format only.
  • Full accessibility and internationalization.

Editing concepts

Editing a book in Obi. On the right the table of contents; on the left the four sections with their audio strips. The last section and its last audio block are currently selected.

This screenshot (click on the image to enlarge) shows the workspace in Obi. At its core, Obi is a structured audio editor; it is designed to manage phrases, which are the smallest navigation units in the book. As an atomic element, a phrase is represented by a block in the Obi workspace.

DAISY books have a structure which is called ``table of contents'' (or TOC) in Obi. The TOC is hierarchical and is represented as a tree view on the left side of the workspace. A node in this tree is a section. New sections and subsections can be added and moved around in the tree view.

Each section has contents, which are the aforementioned phrases. The audio blocks for the phrases are arranged in strips which can be seen in the strip view on the right side of the workspace. Blocks in strips can be moved, copied, split and merged. New audio content is added either by importing existing audio files, or recording new audio directly from Obi.

A complete manual is distributed with Obi and is accessible in the application through the Help menu, or by pressing F1 at any time.

Download

Obi 0.5 is available for download for Microsoft Windows (currently tested under Windows XP only.) You will need the .Net framework 2.0 as well as Direct X 9 to make it run. Important note: this is an early release that still contains many bugs and misses a lot of features. Projects edited with this version are not guaranteed to be editable in future versions; integrity of data is not guaranteed either. Release notes

Known problems, bugs and features (or lack thereof) in version 0.5 include:

  • Keyboard navigation in the strip view uses the tab key only; it will use arrow keys instead.
  • Some shortcuts are not working in some situation, or are wrong. Be careful with the delete key :(
  • No export to DAISY yet. Obi uses the Urakawa toolkit XUK format (XML for Urakawa) which can then be transformed to DAISY 2 or 3. Obi 1.0 will feature native DAISY 3 export.
  • No QA player. Only single audio blocks can be played back.
  • No transport bar either yet.
  • No VU meter for playing and recording. This will be added in version 1.0, along with the ability to change the recording volume.
  • The application itself is English-only, but there is of course no restriction to the language of the audio. Unicode text is OK, but there is no right-to-left support yet.

Missing features listed here are currently planned for inclusion by version 1.0, unless otherwise noted.

Roadmap

The next three releases of Obi are: a more stable alpha version (0.6) in late September, a beta version (0.9) in January 2007, with version 1.0 in early 2007. Version 1.0 should be available in other languages than English. Future versions will include more media support (text and images), DAISY features (skippable and escapable structures, resources, etc.) and internationalization features (right-to-left language support, more translations.)
