Obi is a DAISY/NISO Digital Talking Book production tool. The Obi software was initially created to demonstrate the capabilities of the Urakawa SDK. Since then, it has evolved into a powerful authoring tool, which is easy to use, accessible and internationalized.

Obi is free software; it is available at no cost and its source code is available for anyone to run, modify and redistribute. Obi is released under the LGPL license.

The intended audience of Obi is the individual DAISY reader who wants to create her own DAISY contents. This includes, but is not restricted to, print-disabled users.

Features

Obi is by design a simple tool with a small feature set:

  • Support for audio-only book editing with a table of contents (in DAISY terms, this means NCX-only DTBs)
  • Support for uncompressed RIFF/WAVE audio format only
  • Full accessibility (keyboard navigation, support for screen readers)
  • Internationalization

Editing concepts


Editing a book in Obi. On the right the table of contents; on the left the four sections with their audio and structure strips. Page 1 is currently selected.

This screenshot (click on the image to enlarge) shows the workspace in Obi. At its core, Obi is a structured audio editor; it is designed to manage phrases, which are the smallest navigation units in the book. As an atomic element, a phrase is represented by a block in the Obi workspace.

DAISY books have a structure which is called “table of contents” (or TOC) in Obi. The TOC is hierarchical and is represented as a tree view on the left side of the workspace. A node in this tree is a section. New sections and subsections can be added and moved around in the tree view.

Each section has contents, which are the aforementioned phrases. The audio blocks for the phrases are arranged in strips which can be seen in the strip view on the right side of the workspace. Blocks in strips can be moved, copied, split and merged. New audio content is added either by importing existing audio files, or recording new audio directly from Obi.

Obi uses its own format to store projects (the format is XML based and called XUK) This means that Obi projects cannot be opened directly in a DAISY player, and conversely, Obi cannot open a random DAISY book for editing. This problem will be addressed by DMFC, which will allow conversion to and from XUK, DAISY 2 and DAISY 3.

A complete manual is distributed with Obi and is accessible in the application through the Help menu, or by pressing F1 at any time.

Release notes

Known problems, bugs and features (or lack thereof) in version 0.5.4.927 include:

  • Keyboard navigation in the strip view uses the tab key only; it will use arrow keys instead.
  • Some shortcuts are not working in some situation, or are wrong. Be careful with the delete key :(
  • Only single audio blocks can be played back; no transport bar.
  • No display of waveforms for audio blocks. The abstract representation of sound is actually a feature, as it allows for a simpler and more accessible interface. However, waveform audio editing (e.g., selecting a chunk of audio with a mouse) is planned for version 1.0 of Obi, but will be an optional feature.
  • No VU meter for playing and recording. This will be added in version 1.0, along with the ability to change the recording volume.
  • The application itself is English-only, but there is of course no restriction to the language of the audio. Unicode text is OK, but there is no right-to-left support yet.
  • Accessibility support is still minimal. Users of screen-readers, braille displays and alternative input devices are welcome to send their feedback.
  • The Obi manual is currently in XHTML format; although we are trying to make it as accessible as possible, it is still text-only. In the future, the manual will be in DAISY format (and editable by Obi, of course.) It will also be more complete.
  • The only structure items are page numbers. The first phrase of every section is understood to be the heading for the section.

Version 1.0 of Obi will include the following features:

  • More DAISY metadata: currently there is only a very simple and incomplete subset (see the Edit metadata menu in Obi.)
  • Indenting strips to reflect the structure in the strips view, and collapsing strips to hide their contents in order to clear up the strip view.
  • Search function: all text contents (section labels, annotations of audio blocks, metadata) will be searchable.
  • Autonumbering of pages.
  • Dropping page markers on the fly while recording.
  • QA playing: more support for audio playing inside Obi, and interfacing with AMIS (http://amis.sf.net/) for playback of books.
  • Toolbar with buttons for most actions.
  • Native DAISY 3 export.
  • Support for invalid assets: when reading a project, if an asset file cannot be found, the project cannot be open. A better behavior is to create an invalid block, that has limited operations, so that the user can set a correct asset again and carry on with editing the book.
  • Opening large projects should bring up a progress bar and hide the project panel until it is ready to be shown.
  • Enabling and disabling phrases and section: a phrase or section can be marked as enabled or disabled. Disabled sections will not be exported or played back in normal QA playing mode, but the data is still available while authoring.
  • Translation to other languages than English.
  • Better color scheme, and more importantly, customizable color schemes with defaults for user with low vision requiring high contrast. The font sizes must be customizable as well to allow for large type.

Roadmap

The next three releases of Obi are:

  • a fuller-featured alpha version (0.7) in mid-November
  • a beta version (0.9) in January 2007
  • version 1.0 in early 2007

Future versions will include more media support (text and images), DAISY features (skippable and escapable structures, resources, etc.) and better internationalization (right-to-left language support, more translations.) We encourage the suggestion of additional features for post-1.0 versions of Obi, but we intend to keep it as simple as possible.

Although the Urakawa toolkit is designed to be portable, Obi is unfortunately Windows-only at the moment and will stay so for the foreseeable future. We hope that Obi will inspire the development of other DAISY production tools for other platforms, and that the Urakawa toolkit will make such development easier.
