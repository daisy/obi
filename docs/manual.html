The work area

Strips

Strips are containers for blocks and map one-to-one with sections. They are composed of:

  • The name of the section (same as shown in the TOC view, changing one changes the other.)

  • The thumbnail of the section image (can be shown full-size in a modal window, and/or in the sidebar Ã  la iTunes?)

Blocks

All blocks belong to one of the following class:

  • Plain audio block: this is the regular kind of audio blocks. They have a wave-form display.

  • Page block: a block with a page number (can be automatic or manually assigned.)

  • Heading block: represents the heading for the section strip that it belongs to.

  • Escapable block: marked as escapable in the SMIL output. (See Z spec section 7.4.1)

  • Skippable block: can be skipped by default.

  • Group block: a group block contains one or more blocks of the same kind which all share the same properties.

  • Also: footnotes and references, producer notes, custom blocks.

All of these blocks can have the following properties:

  • Empty or non-empty: the block has audio data or not. An empty block has no wave-form display, naturally, but has two buttons or links to import or record audio data into it.

  • Used or unused: unused blocks are kept in the project but are not exported and skipped during playback (unless explicitely played.)
